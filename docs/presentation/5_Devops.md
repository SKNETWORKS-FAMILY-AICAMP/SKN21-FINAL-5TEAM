# 5. DevOps & Infrastructure

무중단 라이브 서비스로 배포되기 위한 인프라 관리의 핵심 포인트는 ‘자동화’, ‘격리(Isolation)’, 그리고 '안전한 네트워크 통신'입니다.

## 아키텍처 및 무중단 배포(CI/CD) 기반

- **Docker & Docker Compose 기반 불변 인프라(Immutable Infrastructure)**:
  개발자들의 운영 체제가 달라서 발생할 수 있는 휴먼 에러("내 로컬에서는 되는데")를 원천 차단하기 위해 애플리케이션(Frontend, Backend, MySQL, Nginx) 전체를 컨테이너로 패키징했습니다.
  특히, 마이크로서비스 간의 시작 타이밍으로 인한 크래시(Crash) 방지를 위해 `Health Check` 의존성을 도입하여 무거운 DB 컨테이너가 서빙 준비를 완벽히 마친 뒤에 백엔드가 기동되도록 자동화 제어했습니다.
- **Nginx 리버스 프록시 및 로드 밸런싱 준비**:
  Nginx 컨테이너를 단일 진입점(포트 80/443) 프록시로 활용해 보안(SSL Termination 등)과 라우팅을 전담합니다. 특히 AI 챗봇의 **Server-Sent Events(SSE) 스트리밍** 데이터가 서버 단에서 버퍼링(지연)되지 않도록 `proxy_buffering off` 설정을 통해 사용자 화면에 즉시 토큰이 출력되는 체감 속도를 보장했습니다. (라이브 도메인: `moyeo.kro.kr` 연동 완료)
- **AWS Cloud VPC 기반 격리 배포**:
  클라우드(EC2) 환경에서 구동하되, VPC 가상망을 활용하여 외부에 데이터베이스를 직접 노출시키지 않고 오직 프록시(Nginx)를 통해서만 접근 가능하게 만들며 보안 컴플라이언스를 강화한 인프라 셋업입니다.
