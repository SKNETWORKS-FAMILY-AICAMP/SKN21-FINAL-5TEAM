# 3. Backend Architecture (FastAPI & SQLAlchemy)

대규모 트래픽이 몰리는 이벤트(블랙프라이데이 등)나 복잡한 AI 추론 요청이 몰릴 때 시스템 전체가 다운되는 단일 장애점(SPOF)을 막기 위한 비동기 및 무결성 아키텍처입니다.

## 기술 도입 배경 및 프로덕션 고려사항

- **FastAPI 비동기 논블로킹(Non-blocking) I/O 처리**:
  AI 모델이 복잡한 RAG 검색 및 추론을 수행하느라 외부 API 호출 혹은 DB I/O에서 최대 5~10초의 대기가 발생할 수 있습니다. 동기식 프레임워크를 사용하면 스레드가 블로킹되어 서비스 장애로 직결되나, FastAPI 비동기 처리를 도입하여 한 사용자의 AI 추론 중에도 다른 사용자의 장바구니/주문 요청을 동시에 처리하는 높은 동시성(Concurrency)을 확보했습니다.
- **도메인 복잡도 관리 및 비즈니스 무결성 (SQLAlchemy ORM)**:
  쇼핑몰의 주문 상태(주문 생성 → 결제 완료 → 배송 중 → 환불 등)는 데이터베이스 수준에서 엄격하게 관리되어야 합니다. 소프트 딜리트(Soft-delete), Cascade 삭제 등 비동기 요청 환경에서도 동시성 이슈 없이 트랜잭션을 보장하도록 로직을 ORM 객체에 캡슐화시켰으며, Pydantic을 활용한 철저한 요청/응답 검증으로 런타임 오류 가능성을 원천 차단했습니다.
