## 1. 문서 개요

- 대상 서비스: `ecommerce(MOYEO)`, `bilyeo`, `food`, `chatbot`, `SaaS`
- 기준 소스: 저장소 내 각 서비스 디렉토리 기준
- 작성 기준: 화면/사용자 흐름 + API/어댑터 동작 + 관리자/운영 기능 기준
- 현재 상태:
  - `ecommerce`, `bilyeo`, `food`, `chatbot`은 코드 분석 기반 초안으로 `미실행` 표기
  - `SaaS`는 현재 개발 진행중이므로 `예상 O(검증 필요)` 기준으로 별도 표기

## 2. 테스트 목적

- 사용자 핵심 구매 흐름이 정상 동작하는지 검증
- 주문 후 처리 기능(취소, 환불, 리뷰, 배송조회)의 정상 여부 검증
- 마이페이지 및 배송지/포인트/상품권 기능의 정상 여부 검증
- AI 챗봇 기반 추천/상담 보조 기능의 동작 여부 검증
- 관리자 페이지의 사용자 행동 조회 및 배송 관리 기능 검증
- 저장소 내 개별 서비스 간 역할과 통합 구조를 서비스별로 구분하여 검증
- SaaS Adapter 계층의 멀티사이트 연동 준비 상태를 점검

## 6. 진입/종료 기준

### 테스트 시작 기준

- 프론트엔드와 백엔드가 정상 기동 상태일 것
- 기본 카테고리/상품/옵션/주문/배송지 데이터가 준비되어 있을 것
- 일반 사용자 및 관리자 계정이 생성되어 있을 것
- 테스트 대상 버전이 배포 또는 로컬 환경에 반영되어 있을 것

### 테스트 종료 기준

- 최우선 핵심 시나리오 점검 완료
- 실패 및 보류 항목 식별 완료
- 주요 결함에 대한 원인/영향도 정리 완료
- 결과 요약 및 증빙 자료 정리 완료

## 11. 우선 검증 권장 순서

시연이나 배포 직전에는 아래 순서로 우선 확인하는 것을 권장합니다.

1. 회원가입/로그인/로그아웃
2. 상품 조회 -> 장바구니 -> 결제 -> 주문내역
3. 주문 취소/환불/리뷰
4. 마이페이지(배송지/포인트/상품권)
5. 챗봇 상품추천/주문 CS/이미지 업로드
6. 관리자 히스토리/배송관리

## 14. 비고

- 현재 문서는 실제 테스트 결과표가 아니라, 코드 기준으로 작성한 `테스트 계획 + 결과 입력용 초안`입니다.
- 따라서 `작동 여부`는 실제 실행 후 `O`, `X`, `부분성공`, `미지원` 등으로 갱신하는 것을 권장합니다.
- 상품 상세 경로(`/products/{id}`)는 이동 코드가 존재하므로, 실제 상세 페이지 구현 여부는 별도 확인이 필요합니다.
