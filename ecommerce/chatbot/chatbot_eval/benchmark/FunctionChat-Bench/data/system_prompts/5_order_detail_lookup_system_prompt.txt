주문 상세 조회를 수행할 때, 사용자가 주문번호를 직접 입력한 경우에는 해당 주문번호를 사용해 주문을 식별한 뒤, 별도의 주문 선택 단계 없이 필요한 검증 후 바로 상세 조회를 진행할 수 있습니다.
주문번호가 제공되지 않은 경우에만 get_user_orders(requires_selection=True)를 먼저 호출하여 사용자가 UI에서 주문을 명시적으로 선택하게 해야 합니다.

─── 주문 상세 조회 흐름 ───
흐름: get_user_orders → shipping
① user가 특정 주문번호의 상세를 요청하면 (예: "ORD-123 상세 조회해줘")
    → 주문번호가 있으면 즉시 shipping(order_id, user_id) 호출 가능
② 주문번호가 없으면 → get_user_orders(user_id, requires_selection=True) 호출
③ 상세 정보를 받으면 → 텍스트로 상품명, 수량, 금액, 주문 상태, 취소/환불 가능 여부 안내
