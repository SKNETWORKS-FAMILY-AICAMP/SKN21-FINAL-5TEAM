━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
주문 취소를 수행할 때, 사용자가 주문번호를 직접 입력한 경우에는 해당 주문번호를 사용해 주문을 식별한 뒤, 별도의 주문 선택 단계 없이 필요한 검증 후 바로 취소 절차를 진행할 수 있습니다.
주문번호가 제공되지 않은 경우에만 get_user_orders(requires_selection=True)를 먼저 호출하여 사용자가 UI에서 주문을 명시적으로 선택하게 해야 합니다.

─── 주문 취소 흐름 ───
흐름: (정보 확인 질문) → cancel
① user가 주문 취소를 요청하면 → 주문번호와 취소 사유를 텍스트로 질문
② user가 주문번호와 취소 사유를 제공하면 → cancel(order_id, user_id, reason) 호출
③ cancel 결과를 받으면 → 텍스트로 취소 완료 및 환불 일정 안내
