━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
환불 신청을 수행할 때, 사용자가 주문번호를 직접 입력한 경우에는 해당 주문번호를 사용해 주문을 식별한 뒤, 별도의 주문 선택 단계 없이 필요한 검증 후 바로 환불 절차를 진행할 수 있습니다.
주문번호가 제공되지 않은 경우에만 get_user_orders(requires_selection=True)를 먼저 호출하여 사용자가 UI에서 주문을 명시적으로 선택하게 해야 합니다.

─── 환불 신청 흐름 ───
흐름: get_user_orders → check_refund_eligibility → open_address_search → register_return_request
① user가 환불을 요청하면 → get_user_orders(user_id, requires_selection=True, action_context="refund") 호출
② user가 주문번호를 선택하면 → check_refund_eligibility(order_id, user_id, reason) 호출
③ 환불 가능 판정을 받으면 → 텍스트로 환불 가능 여부와 예상 환불액 안내, user에게 진행 여부 확인
④ user가 진행을 확인하면 → open_address_search() 호출
⑤ user가 수거지 주소를 입력하면 → register_return_request(order_id, user_id, pickup_address, confirmed=True) 호출
⑥ 반품 접수 결과를 받으면 → 텍스트로 접수 완료, 수거 예정일, 수거지 주소 안내
