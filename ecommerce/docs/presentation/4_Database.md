# 4. Database Architecture (MySQL & Qdrant)

데이터의 성격에 따라 데이터베이스 물리/논리 모델을 완전히 분리하여, AI 시스템 부하가 커머스 핵심 결제 과정에 영향을 주지 않도록 격리(Isolation) 아키텍처를 도입했습니다.

## 프로덕션 레벨 데이터베이스 설계 전략

- **Core RDBMS (MySQL 8.0)**:
  결제, 할인 포인트, 재고 등의 커머스 핵심 데이터 처리는 고가용성 보장 및 락(Lock) 관리가 필수적입니다. 복잡한 다중 조인 쿼리도 효율적으로 분산 처리하도록 3NF(제3정규형) 기반으로 테이블 구조를 설계했으며, AI 에이전트의 잘못된 SQL 조작 시도로부터 이 Main DB를 안전하게 격리시켰습니다.
- **Vector DB (Qdrant) 및 비용 최적화**:
  RBMDS에 LLM을 무리하게 연결하지 않고 AI가 참조할 방대한 FAQ, 규정집, 상품 메타데이터는 오직 효율적인 하이브리드 검색(Dense/상대적 의미 + Sparse/키워드)을 위해 Qdrant 벡터 전용 DB로 오프로딩했습니다. 메모리에 상주하는 인덱스 크기를 최적화하여 값비싼 클러스터 비용 지출 없이 안정적으로 수만 건의 문서를 서치하도록 설계했습니다.
