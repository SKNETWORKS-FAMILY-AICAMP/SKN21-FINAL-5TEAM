# 1. 기술 스택 (Tech Stack)

본 프로젝트는 단순한 PoC(개념 증명) 수준을 넘어 대용량 트래픽과 안정적인 AI 추론이 필수적인
실제 프로덕션 레벨의 커머스 환경을 염두에 두고 아키텍처를 설계했습니다. 성능 확보, 비용 절감, 그리고 운영 안정성이라는 3가지 핵심 축을 기준으로 기술 스택을 구성했습니다.

| 분야         | 기술                                 | 채택 사유 (Production 지향)                                                                                                     |
| ------------ | ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------- |
| **AI Core**  | LangGraph, OpenAI API, RAG, **vLLM** | 모델의 환각(Hallucination) 제어를 위한 에이전트 워크플로우 통제 및 **vLLM** 도입을 통한 토큰 처리량 극대화/인퍼런스 비용 최소화 |
| **Backend**  | Python 3.13, FastAPI, SQLAlchemy     | 다수 사용자의 AI 대기 시간(TTFT) 병목을 해소하기 위한 비동기(Async) 런타임 최적화                                               |
| **Frontend** | Next.js 16, React 19, Tailwind CSS   | 방대한 상품 이미지 및 동적 챗봇 UI 렌더링 부하를 줄이기 위한 SSR 처리 기법 및 실시간 SSE 스트리밍                               |
| **Database** | MySQL 8.0, Qdrant                    | 민감한 결제/주문 데이터 무결성을 보장하는 RDBMS와 벡터 쿼리(RAG) 전용 고성능 DB의 분리 운영(Isolation)                          |
| **DevOps**   | Docker, Nginx, AWS, Git              | CI/CD 및 무중단 배포를 고려한 컨테이너 격리 및 로드밸런싱 인프라 구성                                                           |
