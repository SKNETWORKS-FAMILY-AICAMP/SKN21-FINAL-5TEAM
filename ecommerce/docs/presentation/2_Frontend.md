# 2. Frontend Architecture (Next.js)

고객 접점(B2C) 쇼핑몰 프론트엔드로서, 사용자 체감 속도(LCP) 향상과 실시간 챗봇 인터페이스의 끊김 없는 경험 제공에 집중했습니다.

## 기술 도입 배경 및 프로덕션 고려사항

- **초고속 페이지 렌더링 및 이미지 최적화**:
  이커머스의 핵심 자산인 고해상도 상품 이미지가 다수 맵핑되는 환경에서 대역폭 비용과 로딩 속도의 Trade-off가 발생합니다. 이를 Next.js의 내장된 이미지 최적화(`next/image`)와 서버 사이드 렌더링(SSR)을 결합하여 클라이언트 렌더링 속도와 네트워크 비용을 대폭 절감했습니다.
- **실시간 스트리밍 UI 구조 및 메모리 관리**:
  단순 텍스트 챗봇과 달리, 상품 카드와 주문 리스트 등의 무거운 컴포넌트가 대화창(Bubble) 내에 지속적으로 쌓이는 형태입니다. DOM 노드 비대로 인한 클라이언트 브라우저 렉(Lag)을 방지하도록 상태 관리를 최적화했으며, 서버로부터 전달되는 AI의 응답을 Server-Sent Events(SSE) 스트리밍 형식으로 받아 레이턴시(Latency)를 극적으로 단축시켰습니다.
