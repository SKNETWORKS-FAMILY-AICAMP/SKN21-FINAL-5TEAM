# 9. 프로젝트 기능 시연 (Demo)

단순한 API 래퍼(Wrapper) 챗봇이 아닙니다. 이커머스 백엔드의 트랜잭션, 상품 렌더링 서버, 그리고 AI 서빙 아키텍처가 유기적으로 실시간 통신하는 프로덕션 환경의 E2E 시연입니다.

---

## 시연 시나리오 1: 스트리밍 렌더링과 UI 제어

**"AI 응답 지연(Latency) 이슈를 해결하고, 대화창에 무거운 커머스 UI 뷰를 띄웁니다."**

- **작업**: 사용자가 "최근 주문한 내역 좀 보여줘"라고 입력.
- **아키텍처 포인트**:
  - FastAPI 백엔드가 AI(LangGraph)의 상태를 비동기로 대기하는 동안 브라우저가 정지되지 않습니다.
  - 모델의 함수 호출(`get_order_list`) 결과가 반환되는 즉시, Next.js 프론트엔드로 `SSE(Server-Sent Events)` 청크 형태의 UI_ACTION 페이로드를 스트리밍합니다.
  - 결과적으로 로딩 스피너 대신 실시간 텍스트 출력과 함께 **무거운 주문 객체 정보가 담긴 장바구니/주문서 React 컴포넌트**가 동적으로 부드럽게 렌더링됩니다.

## 시연 시나리오 2: Qdrant 기반 하이브리드 RAG와 안전망

**"오답(할루시네이션)을 내뱉기보다 정확한 Reference를 우선순위화 합니다."**

- **작업**: 사용자가 "저기요, 반품할 때 배송비는 어떻게 돼?"라고 질문.
- **아키텍처 포인트**:
  - 단순 LLM 지식에 의존하지 않고, 즉시 Qdrant 벡터DB에 쿼리하여 FAQ/약관 문서 중 '반품 배송비'에 대한 Sparse/Dense 하이브리드 검색을 병렬 수행합니다.
  - 교차 검증된(Cross-Encoder Reranking) 문맥 베이스 위에서만 답변이 생성되어, 회사 규정에 위반되는 금전적 안내 오류(Legal Risk)를 원천 차단하는 과정을 보여줍니다.

## 시연 시나리오 3: 에이전틱 트랜잭션 및 강제 승인(Human-in-the-Loop)

**"서비스 환경에서 가장 치명적인 '의도치 않은 데이터 삭제/변경'을 방어합니다."**

- **작업**: 사용자가 무턱대고 "어제 시킨 거 그냥 다 취소해 줘"라고 막연하게 지시.
- **아키텍처 포인트**:
  - 주문 상태 변경은 SQLAlchemy 상에서 '결제 취소 API'를 건드리는 민감한(Sensitive) DB 트랜잭션입니다. AI가 이 툴(Tool)을 호출하려 시도합니다.
  - 이때 LangGraph 엔진 내부의 **Smart Validation Node**가 개입, 어떤 주문인지 파라미터가 없음을 감지하고 역으로 사용자에게 "어떤 주문을 취소하시겠습니까?"를 선택하게 리디렉션합니다.
  - 사용자가 대상을 선택하더라도, **Human Approval Node(인간 최종 승인 단계)**가 강제로 "정말 [상품명]을 취소할까요?"라며 통제권을 뺏어오는 프로덕션 레벨의 안전망(Guardrail)이 작동하는 것을 시연합니다.
