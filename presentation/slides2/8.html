Onboarding Pipeline

08

사이트 분석부터 patch 생성까지 자동화

온보딩은 코드 생성 하나로 끝나는 도구가 아니라, 분석·생성·검증·복구를 포함하는 자동화 파이프라인입니다.

01

Analyze

프레임워크, 인증, API, mount 지점 추론

02

Map

수정 후보, 라우트, 삽입 포인트 탐색

03

Plan / Generate

어댑터, 위젯, patch 산출물 생성

04

Runtime Validate

별도 workspace에서 적용 시뮬레이션

05

Evaluate / Smoke

frontend/backend evaluator와 smoke test

06

Recover / Export

recovery loop 이후 승인된 patch export

Execution Roles

Analyzer → Planner → Generator → Diagnostician

발표에서는 메인 파이프라인을 먼저 보여주고, 실행 주체는 보조 설명으로 짧게 연결합니다.

Key Message

생성 후 끝나는 것이 아니라
검증 후 export

runtime validate, smoke test, recovery loop를 포함한 자동화라는 점이 핵심 차별점입니다.
