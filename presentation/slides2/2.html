Project Background

02

반복 상담은 많지만,
사이트 적용은 느리다

고객 문의의 반복성과 사이트별 연동 비용 문제를 동시에 해결하는 방향으로 프로젝트를 기획했습니다.

Customer Problem

고객 문의는 반복적입니다

  • 사이즈, 배송, 교환/환불, 상품 비교 문의가 반복됩니다
  • 의류 도메인은 현재 상품 맥락이 들어가야 답변 품질이 올라갑니다

Operator Problem

도입 자체가 느립니다

  • 사이트마다 프론트 구조와 인증 방식이 달라 연동 비용이 큽니다
  • 탑재 과정이 수작업이면 챗봇이 좋아도 재사용성이 낮아집니다

Project Goal

상담 품질 + 도입 속도

도메인 특화 상담과 사이트별 온보딩 자동화를 함께 해결합니다.

의류 특화 상담 인증 기반 기능 분리 온보딩 자동화
