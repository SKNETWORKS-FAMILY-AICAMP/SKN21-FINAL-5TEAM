Onboarding Operations

09

자동 생성이 아니라,
승인·검증·재시도를 갖춘 운영형 시스템

생성 결과를 바로 반영하지 않고, 승인 게이트와 이벤트 추적을 둬서 운영 가능한 형태로 만들었습니다.

Gate 1

Analysis

대상 사이트 분석 결과를 먼저 검토해 초기 추론 오류를 차단합니다.

Gate 2

Apply

실제 수정 적용 이전에 검증 단계를 두어 위험한 산출물을 걸러냅니다.

Gate 3

Export

최종 승인된 결과만 patch로 export해 배포 가능한 상태로 넘깁니다.

State

Redis

job과 event를 저장해 장시간 실행 상태를 추적합니다.

Streaming

SSE

실행 단계와 승인 상태를 실시간으로 구독할 수 있습니다.

Bridge

Slack

외부 운영 채널에서 승인과 상태 확인이 가능합니다.

Recovery

Resume

실패 시 진단과 재시도를 이어가며 실행을 복구합니다.
